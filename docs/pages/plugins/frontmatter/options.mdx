# Options

<Callout type="info">
  Please view options exposed by [typedoc-plugin-markdown](/docs/options) in addition to those listed here.
</Callout>

## frontmatterGlobals

<Callout emoji="ðŸ’¡">Specify static variables to be added to all frontmatter blocks.</Callout>

> Accepts a key/value object.


```yaml filename="YAML"
---
layout: docs
sidebar: true
---
```


```json filename="typedoc.json"
{
  "frontmatterGlobals": {
    "layout": "docs",
    "sidebar": true
  }
}

```

## frontmatterCommentTags

<Callout emoji="ðŸ’¡">Specify which comment block tags should be added to frontmatter.</Callout>

> Accepts an Array.

Frontmatter variables can be added by extracting comments from block (@) tags.

Please note tags must be added to the comment blocks of the symbol exported to a page.

 ```ansi filename="Block Tags (someModule.ts)"
 /**
 * @author Joe Bloggs
 *
 * @description A description that will be added to frontmatter.
 */
 ```

```yaml filename="YAML (someModule.md)"
---
author: Joe Bloggs
description: A description that will be added to frontmatter.
---
```


```json filename="typedoc.json"
{
  "frontmatterCommentTags": [
    "author",
    "description"
  ]
}

```

___

## preserveFrontmatterCommentTags

```json filename="typedoc.json"

}

```

## frontmatterNamingConvention

<Callout emoji="ðŸ’¡">The naming convention that variables should be output as. </Callout>

> Accepts either `"camelCase"` or `"snakeCase"`. Defaults to `"camelCase"`.

Block tags have to be written in camelCase (see [tsdoc.org](https://tsdoc.org/pages/spec/tag_kinds)).

This option can configure the output style of frontmatter variables when written to YAML.


```json filename="typedoc.json"
{
  "frontmatterNamingConvention": "camelCase"
}

```
