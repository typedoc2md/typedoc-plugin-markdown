# Output File Structure

TypeDoc generates documentation models according to exports derived from the given [entryPoints](https://typedoc.org/documents/Options.Input.html#entrypoints) and [entryPointsStrategy](https://typedoc.org/documents/Options.Input.html#entrypointstrategy) TypeDoc configurations.

TypeDoc 0.28.0 introduced an additional [router](https://typedoc.org/documents/Options.Output.html#router) option which can be used to modify the output folder structure.

This plugin now utilises this mechanism for output file generation.

## Built-In Plugin Routers

The plugin ships with the following custom built-in routers:

### "member" (default)

```json filename="typedoc.json"
{
  "router": "member"
}
```

_Note: This is equivalent to the deprecated `outputFileStrategy="members"`._

Generates an individual file for each exported module member kind. Pages are grouped by kind within their respective module hierarchy.

This mirrors a combination of TypeDoc's `kind` and `structure` routers.

This is the default router of the plugin.

<FileTree>
  <FileTree.File name="README.md" />
  <FileTree.Folder name="module-a" defaultOpen>
    <FileTree.Folder name="classes" defaultOpen>
      <FileTree.File name="ClassA.md" />
      <FileTree.File name="ClassB.md" />
    </FileTree.Folder>
    <FileTree.Folder name="functions" defaultOpen>
      <FileTree.File name="FunctionA.md" />
      <FileTree.File name="FunctionB.md" />
    </FileTree.Folder>
  </FileTree.Folder>
  <FileTree.Folder name="module-b" defaultOpen>
    <FileTree.Folder name="classes" defaultOpen>
      <FileTree.File name="ClassA.md" />
      <FileTree.File name="ClassB.md" />
    </FileTree.Folder>
  </FileTree.Folder>
</FileTree>

### "member-flat"

```json filename="typedoc.json"
{
  "router": "member-flat"
}
```

_Note: This is equivalent to the deprecated `outputFileStrategy="members"` and `flattenOutputFiles=true`._

Like "member", but flattens output to a single directory.

<FileTree>
  <FileTree.File name="README.md" />
  <FileTree.File name="module-a.Class.ClassA.md" />
  <FileTree.File name="module-a.Class.ClassB.md" />
  <FileTree.File name="module-a.Function.FunctionA.md" />
  <FileTree.File name="module-a.Function.FunctionB.md" />
  <FileTree.File name="module-b.Class.ClassA.md" />
  <FileTree.File name="module-b.Class.ClassB.md" />
</FileTree>

### "module"

```json filename="typedoc.json"
{
  "router": "module"
}
```

_Note: This is equivalent to the deprecated `outputFileStrategy="modules"`._

Generates a single file for every module and namespace where all module member kinds are hoisted to a single file.

This creates a flat navigation structure and reduces the amount of files generated.

<FileTree>
  <FileTree.File name="README.md" />
  <FileTree.File name="module-a.md" />
  <FileTree.File name="module-b.md" />
</FileTree>

### "module-flat"

```json filename="typedoc.json"
{
  "router": "module-flat"
}
```

_Note: This is equivalent to the deprecated `outputFileStrategy="modules"` and `flattenOutputFiles=true`._

Like "module", but flattens output to a single directory.

<FileTree>
  <FileTree.File name="README.md" />
  <FileTree.File name="package-1.module-a.md" />
  <FileTree.File name="package-1.module-b.md" />
  <FileTree.File name="package-2.module-a.md" />
  <FileTree.File name="package-2.module-b.md" />
</FileTree>

## Custom Routers

For the adventurous, you can extend the routers to a custom output.

### Extend Custom Router

```json filename="custom-router.js"
{
  "plugin": ["./custom-router.mjs"],
  "router": "custom-router"
}
```

```js filename="custom-router.mjs"
import { ReflectionKind } from 'typedoc';
import { MemberRouter } from 'typedoc-plugin-markdown';

export function load(app) {
  app.renderer.defineRouter('custom-router', CustomRouter);
}

export class CustomRouter extends MemberRouter {
  relativeUrl(from, to) {
    return `wiki/${super.relativeUrl(from, to)}`;
  }
}
```

## TypeDoc Core Routers

```json filename="typedoc.json"
{
  "router": "kind"
}
```

TypeDoc ships with several built-in routers that can be be used with this plugin.
This provides the ability to generate output files in the same structure as the default HTML theme but rendered in Markdown.

Available routers are **"kind"**, **"kind-dir"**, **"structure"** ,**"structure-dir"**, **"group"**, **"category"**.

Please see [router](https://typedoc.org/documents/Options.Output.html#router) documentation for details.

The routers are decorated to additionally support the file options of this plugin.
